# 작업 규칙 (CLAUDE.md)

이 프로젝트에서 작업할 때 아래 규칙을 지킨다. (사용자 = 김형래, 제조기술 파트. 회사 제품·공정 지식은 있으나 코딩/엔지니어링은 새내기이므로, 쉽게 설명하고 방향을 잡아주며 결과를 반드시 증명한다.)

## 문서·버전 관리
- 프로젝트 구조는 `PROJECT_OVERVIEW.md`에 종합한다.
- `WORKLOG.md`를 작업 단위마다 갱신 → 커밋 → `origin/main` 푸시한다. (P0~P3 백로그, 결정 대기 목록, 결정 사항 기록)
- 공개 저장소이므로 라이선스 알고리즘 등 민감 정보는 문서에 기재하지 않는다.

## 진행 방식
1. **계획 먼저** — 큰 기능/변경은 바로 코딩하지 말고, 계획을 제시하고 사용자 승인을 받은 뒤 실행한다.
2. **작게 쪼개서 단계로** 진행하고, 단계마다 커밋한다.
3. **명백한 버그**는 질문 없이 자동 수정(✅), **방향 결정**이 필요한 것은 ❓로 모아 질문한다.
4. **애매하면 넘겨짚지 말고 먼저 물어본다.**
5. 전문 에이전트를 역할별로 나눠(탐색/구현/검토 등) 병렬 활용한다.

## 검증 (필수)
- "완성했다"고 말하기 전에 **반드시 실행해서 증명**한다. (테스트 실행 결과, 또는 브라우저로 화면을 띄워 확인)
- 화면이 있는 결과물(대시보드 등)은 눈으로 확인 가능한 형태로 보여준다.

## 소통
- 제안·설명은 **짧고 간결하게**. 긴 표·에세이 대신 핵심과 추천안 위주로.
